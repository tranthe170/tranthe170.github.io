Prepare AWS Glue

Create a Database in AWS Glue

Access the AWS Glue Console

  1. Search for and select “Glue”:
    • In the AWS Management Console, use the search bar at the top to search for “Glue.”
    • Click on “Glue” in the search results to open the AWS Glue service.

Create a Database

  1. In the left-hand menu, click on “Databases”:

    • Under the “Data catalog” section, click on “Databases.”

  2. Click on “Add database”:

    • Click the “Add database” button to start creating a new database.

  3. Database name:

    • Enter a name for your database (e.g., my_glue_database) and click “Create.”

Create AWS Glue Roles

Create AWSGlueServiceRoleDefault Role

  1. Go to the IAM Console:

    • Use the AWS Management Console to navigate to the IAM service.

  2. Click on “Roles”:

    • In the left-hand menu, click on “Roles.”

  3. Click “Create role”:

    • Click the “Create role” button to start the process.

  4. Select type of trusted entity and choose use case “Glue”:

    • Choose “AWS service.” Then, select “Glue” as the use case.

  5. Attach policies:

    • Click “Next” to proceed to the permissions step. Search for and select “AWSGlueServiceRole.”

  6. Role name:

    • Click “Next.” Enter AWSGlueServiceRoleDefault as the role name.

  7. Click “Create role”:

    • Click the “Create role” button to finalize.

Create AWSGlueServiceNotebookRoleDefault Role

To properly run AWS Glue notebooks on a Spark cluster, you need to create a specific IAM role (AWSGlueServiceNotebookRoleDefault) with appropriate permissions. This role grants permissions to the notebook, but the actual data processing is handled by the Spark cluster. Therefore, you must allow the notebook to pass its permissions to the Spark cluster using the PassRole permission.

  1. Create another role following the same steps:

    • Repeat the process outlined above to create a new role, naming it AWSGlueServiceNotebookRoleDefault.

  2. Add PassRole permission:

    • Select the AWSGlueServiceNotebookRoleDefault role you just created and click “Create inline policy.”
    • Switch to the JSON view and paste the following code:
    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "iam:PassRole",
      "Resource": "arn:aws:iam::YOUR_ACCOUNT_ID:role/AWSGlueServiceNotebookRoleDefault"
    }
  ]
}
    • Replace YOUR_ACCOUNT_ID with your own AWS Account ID.

  3. Name and create the policy:

    • Enter PassRolePermission as the policy name.

  4. Finalize the role:

    • Click “Create role” to complete the process.

Create a Crawler in AWS Glue

Access the Crawlers Section

  1. In the AWS Glue console, click on “Crawlers”:

    • In the left-hand menu, select “Crawlers.”

  2. Click “Add crawler”:

    • Click the “Add crawler” button to begin the setup.

Configure the Crawler

  1. Name:

    • Enter a name for your crawler (e.g., my_crawler).

  2. Data source:

    • Select “Add a data source.”

  3. Include path:

    • Choose “S3” as the data source. Enter the S3 bucket path where your data is stored (e.g., s3://my-data-bucket/).

  4. IAM Role:

    • Select the AWSGlueServiceRoleDefault role created earlier.

  5. Output:

    • Choose the database you created earlier (e.g., my_glue_database). Enter university_ranking_ for the prefix. Set the Crawler schedule to “On demand.”

  6. Review and click “Finish”:

    • Review your configuration and click “Finish” to create the crawler.

Run the Crawler

  1. Select the crawler:

    • In the Crawlers section, select the crawler you just created.

  2. Click “Run crawler”:

    • Click the “Run crawler” button to start the process.

  3. Wait for the crawler to finish running:

    • The crawler will catalog the data in your S3 bucket.

Create a Glue Notebook

Access the Notebooks Section

  1. In the AWS Glue console, click on “Notebooks”:

    • In the left-hand menu, click on “Notebooks.”

  2. Click “Notebook”:

    • Click the “Notebook” button to start setting up a new notebook.

Configure the Notebook

  1. Configure the notebook:

    • Choose “Start fresh,” select the AWSGlueServiceNotebookRoleDefault role, and click “Create notebook.”

  2. Rename the notebook:

    • Enter “university_ranking_nb” as the notebook name.

  3. Click “Save”:

    • Click the “Save” button to finalize.

You can now write and execute Spark and Python code to process and analyze your data in the Glue notebook.
